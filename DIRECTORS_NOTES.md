## Current Canon

### Project thesis
Teach a small local language model to ground musician-native semantics (phase locking, groove, brittleness, interference, etc.) in a bounded synthetic world where those semantics are computable from state. Evaluate on **intervention grounding** ŌĆö does the model's suggested action actually move the sim the way it claimed?

### Toy world ŌĆö Coupled Oscillator Garden (v0)
- `N = 8` nodes in the unit square.
- Kuramoto-style phase coupling with distance kernel `KßĄóŌ▒╝ = KŌéĆ ┬Ę exp(ŌłÆdßĄóŌ▒╝/Žā)`.
- Optional white-noise perturbation `╬ŠßĄó(t)` with amplitude `╬Ę`.
- Per-node fields: `pos, ŽēŌéĆ, ╬Ė, A, ╬│, voice`.
- Audio is **pulse-driven**: each phase-zero crossing fires a percussive grain with a fixed-pitch voice; stereo pan from `pos.x`. Audio-rate tone coupling is deferred to v0.1.
- Scheduled (not real-time) events: `nudge`, `impulse`, `move`, `setK`.

### Run exports
`audio.wav` (22050 Hz stereo), `state.npz` (`╬Ė`, `A` at 200 Hz), `events.jsonl`, `topology.json`, `labels.json`.

### Ontology (13 labels, multi-label, detector-grounded)
`phase_locked`, `drifting`, `polyrhythmic`, `phase_beating`, `dominant_cluster`, `brittle_lock`, `unstable_bridge`, `groove`, `groove_collapse`, `beat_bloom`, `tension_break`, `flam`, `regime_change`. Each has a deterministic detector; labels are ground truth, not model outputs. Detector confidence is **evidence margin** (how strongly / stably the defining condition was met), not epistemic uncertainty.

### Data pipeline
Regime-template sampler (`LOCKED`, `DRIFTING`, `TWO_CLUSTER`, `BRITTLE`, `SWEEP`, `PERTURBED`, `COLLAPSE`). v0 target: ~5 000 clips ├Ś ~5 tasks = ~25 k examples.

### Task suite
T1 caption, T2 semantic QA, T3 compare, T4 anomaly, T5 explain change, T6 suggest intervention. T6 is the headline evaluable task.

### Model stack (planned)
- Trained encoders: small CNN (or adapter over frozen small CLAP) on mel spectrograms; small transformer on per-node state; topology as concat tokens (no GNN in v0).
- Frozen base LM (Qwen2.5-0.5B or Llama-3.2-1B) + LoRA.
- SFT only for v0; auxiliary losses only if SFT plateaus.

### Evaluation tiers
L1 label accuracy ŌåÆ L2 noise / perturbation robustness ŌåÆ L3 intervention grounding against random + heuristic baselines. Generalization probes across `N`, voice palette, held-out regime.

### Out of scope for v0
Real-time interaction, audio-rate tone coupling, GNNs, RLHF, third-party music datasets, human listening studies.

## Resolved Dragons and Pivots

### 2026-04-20 ŌĆö Claude Opus 4.7
Initial design pass for v0. Key pivots made while drafting:

- **Pulse-driven audio over audio-rate tone coupling.** The obvious starting assumption was continuous tones with Hz-scale beating. Chose pulses instead because (a) it's cheaper to simulate, (b) emergent *rhythm* is the strongest musical phenomenon at the timescale of short clips, and (c) beating can be approximated at the perceptual level by tight near-unison pulses (the `flam` label). Audio-rate tone coupling reconsidered in v0.1.
- **No GNN in v0.** Tempting but unjustified at `N = 8` with known geometry. Concat tokens with positional embeds are sufficient. Revisit only if topology reasoning visibly under-performs.
- **First milestone is data, not model.** Initial draft had a minimal end-to-end model as the first milestone. Reversed: detector definitions are the hardest part of the project and must be sniff-tested against human ears before any model training is meaningful.
- **Intervention grounding as headline eval.** Chosen over label-F1 because label accuracy can be gamed by surface features; running the model's suggestion through the sim and measuring its effect is the only test that distinguishes grounded understanding from pattern matching.
- **Frozen LM + LoRA over full fine-tune.** Single-researcher scale; full fine-tune of even a 0.5B model is overkill for the data budget and loses generality.

### 2026-04-20 ŌĆö Claude Opus 4.7 (doc pass 2)
Terminology tightening for the pulse-first v0 world. No scope changes.

- **`beating` ŌåÆ `phase_beating`.** The original name silently inherits acoustic-wave semantics (Hz-scale amplitude modulation between continuous tones). v0 is pulse-driven; what the detector measures is slow periodic drift between pulse-arrival times of near-frequency streams. Renamed to name the actual quantity. Old name kept in parentheses in the ontology table so the drift is explicit.
- **`interference_bloom` ŌåÆ `beat_bloom`.** Same audio-rate overclaim as above; "interference" strongly suggests wave-level superposition. The phenomenon is a transient excursion of the `phase_beating` index ŌĆö two streams briefly approach coherence and fall back out. `beat_bloom` ties it to its steady-state sibling and drops the misleading connotation.
- **`unstable_bridge` ŌĆö name kept, detector redefined.** Old wording assumed a discrete edge set ("edge whose removal splits the graph"); the garden is distance-weighted, fully connected with no discrete edges. Redefined as a **load-bearing node** whose in-sim ablation drops the cluster's order parameter below the lock threshold. Name kept because the musical "bridge about to collapse" connotation is worth preserving.
- **Detector confidence doctrine recorded.** Confidence in `labels.json` is evidence margin (how strongly / stably a condition was met), not epistemic uncertainty. Detectors are deterministic; graded scores exist so downstream tasks can distinguish marginal from solid.
- **README promoted to a proper front door.** Public-facing summary surfacing the project thesis, the v0 shape, and ŌĆö most prominently ŌĆö intervention grounding as the headline evaluation principle.

### 2026-04-20 ŌĆö Claude Opus 4.7 (doc pass 11 ŌĆö first semantic detectors)
First actual semantic-judgment layer. `sim/detectors.py` lands with two sustained-condition detectors (`phase_locked`, `drifting`) built on top of `kuramoto_order` + `sustained_windows`. Still no detector registry, no YAML thresholds, no `labels.json` schema, no `sim.detect` orchestration ŌĆö just the two functions and their uniform result shape.

- **Module placement.** `sim/detectors.py` rather than `sim/detect.py` or a `sim/detectors/` package. Rationale: plural name flags that this will grow into a module with many small functions; a package is premature at two detectors. Not re-exported from `sim/__init__.py` ŌĆö follows the `sim.derived` convention; callers import from `sim.detectors` explicitly. Keeps the package surface at `load, validate, simulate, ConfigError, SCHEMA_VERSION`.
- **Uniform result shape: `DetectionResult(fired, windows, confidence, longest_window_frames)`.** One NamedTuple shared by both detectors. Fields chosen to be the minimum that (a) tests can assert on cleanly, (b) a future `labels.json` writer can consume without reshaping, (c) is self-explanatory at the REPL. `windows` is a `list[tuple[int, int]]` in the half-open frame convention inherited from `sustained_windows` ŌĆö the union of all detector windows in the project will use this convention. `longest_window_frames` is redundant with `windows` but is the field tests keep wanting, so it lives inline.
- **Threshold policy: hardcoded module constants, not function parameters, not YAML.** Constants (`R_LOCK = 0.9`, `PHASE_LOCKED_MIN_S = 1.0`, `PHASE_LOCKED_HYST_S = 0.1`, `R_DRIFT = 0.5`, `DRIFTING_MIN_S = 2.0`, `DRIFTING_HYST_S = 0.25`) sit at module scope, one block per detector. This is the deliberate first-pass compromise: explicit and grep-findable in code, not hidden behind YAML bureaucracy, not parameterized into the function signature (which would add a tuning surface that tests would have to exercise). The tests import the two `*_MIN_S` constants directly so the "at least this long" assertions are driven by the same number the detector uses ŌĆö tighter coupling than YAML would give, intentional at this scale. `DESIGN_V0.md ┬¦10.7`'s versioned YAML threshold layer is deferred until 4+ detectors exist.
- **Threshold values chosen.** `R_LOCK = 0.9` matches the regime-smoke test's `mean r > 0.9` bar on LOCKED. `R_DRIFT = 0.5` matches the window threshold validated in doc pass 9's DRIFTING sustained-window test ŌĆö already empirically shown to cleanly separate all three regimes. `PHASE_LOCKED_MIN_S = 1.0` is the shortest sustained window that rejects two_cluster's brief-but-high r excursions (max run of `r >= 0.9` on two_cluster state is well below 1 s). `DRIFTING_MIN_S = 2.0` is the value doc pass 9 already validated as the separator (two_cluster's longest low-r valley is 0.43 s). Hysteresis set just long enough to absorb the dominant numerical timescale in each regime ŌĆö 0.1 s for locked (fast numerical jitter), 0.25 s for drifting (slow drift-induced upward spikes).
- **Confidence doctrine applied: evidence margin, not probability.** `phase_locked` ŌåÆ `mean(r ŌłÆ R_LOCK)` across firing-window frames. `drifting` ŌåÆ `mean(R_DRIFT ŌłÆ r)` across firing-window frames. Hysteresis frames are *included* in the mean so weak-but-qualifying windows return small margins ŌĆö the doctrine ("confidence is evidence margin") requires this. Observed on the fixtures: LOCKED ŌåÆ `phase_locked` confidence = 0.0937 (mean `r Ōēł 0.994` in-window); DRIFTING ŌåÆ `drifting` confidence = 0.1842 (mean `r Ōēł 0.316` in-window). `confidence = 0.0` when `fired is False` ŌĆö invariant, cheap to assert in tests, distinguishes silent from weak-fire. ### 2026-04-20 ŌĆö Claude Opus 4.7 (doc pass 10 ŌĆö k-way cluster-assignment primitive)
Generalized the k=2 `split_by_largest_velocity_gap` into a proper reusable k-way partitioner and removed the now-redundant special case. The helper stays narrow: 1D velocity-gap split only, no phase-based sub-clustering, no auto-selection of `k`. Still no detectors, no thresholds YAML, no label schema.

- **`cluster_assignments(mean_vel, n_clusters) -> ClusterAssignments` in `sim/derived.py`.** Given a `(N,)` array of per-node mean phase velocities and a target `n_clusters Ōłł [1, N]`, sorts nodes by velocity and splits at the `n_clusters - 1` largest adjacent gaps. Returns a `ClusterAssignments` NamedTuple with three fields: `labels` (`(N,)` int, cluster id per node), `members` (k-tuple of int arrays sorted by node index), and `separability_ratio` (`min(selected_gaps) / max(unselected_gaps)`, with `+inf` / `0.0` at the obvious degenerate boundaries). Clusters are **ordered ascending by mean-velocity centroid** ŌĆö `members[0]` is the slowest group, `members[k-1]` the fastest ŌĆö so cluster identity is stable across calls on the same input.
- **Signal chosen: per-node mean phase velocity.** At `N = 8` with coupled-oscillator physics, coherent groups share a common collective rotation rate, so a 1D gap-based split on mean velocity is the honest minimum. Alternatives considered and rejected: (a) phase-only clustering ŌĆö fails when clusters have different rotation rates (the `two_cluster` case); (b) full feature-vector k-means over `(velocity, sin ╬Ė╠ä, cos ╬Ė╠ä)` ŌĆö pulls in scipy/sklearn-grade machinery for a problem that's demonstrably 1D at the current scale; (c) hierarchical / DBSCAN variants ŌĆö same overkill objection, and they all still need a distance metric the caller would have to justify. The honest minimum is one scalar per node and a sort. Phase-based sub-clustering inside a velocity-homogeneous group (e.g. anti-phase at the same rotation rate) is left as a deferred extension ŌĆö no current or near-term detector needs it.
- **Determinism & tie-breaking.** Node sorting uses `np.argsort(kind="stable")` so equal velocities preserve node-index order. Gap selection uses `np.argsort(-gaps, kind="stable")` then takes the first `n_clusters - 1` indices ŌĆö stable descending sort means equal-magnitude gaps break ties in favor of the smaller sorted-velocity position (the earlier / slower boundary wins). Near-identical gaps are, as with any gap heuristic, sensitive to input precision ŌĆö this is documented and intrinsic, not a bug to design around. `separability_ratio` is `+inf` when `n_clusters == 1` (no boundaries to weaken) or when `n_clusters == N` and any selected gap is positive, and `0.0` when all velocities coincide.
- **Removed `split_by_largest_velocity_gap` and `VelocitySplit`.** The k=2 case is now `cluster_assignments(v, 2)`: `members[0]` replaces `.low_indices`, `members[1]` replaces `.high_indices`, `separability_ratio` replaces `.bimodal_ratio` with identical semantics for k=2. The old `gap` field had no callers and is dropped (callers who want it can compute `mean_vel[members[1]].min() - mean_vel[members[0]].max()`). No deprecation shim ŌĆö only one caller existed (the two_cluster test) and it's refactored in the same pass.
- **Test refactor is implementation-only.** `tests/test_regimes.py::test_two_cluster_regime_shows_bimodal_structure` now uses `cluster_assignments(v_per_node, n_clusters=2)` and reads `members`, `separability_ratio`. ### 2026-04-20 ŌĆö Claude Opus 4.7 (doc pass 9 ŌĆö sustained-window primitive + DRIFTING sanity test)
Closed the simulator sanity triangle (`locked`, `two_cluster`, now `drifting`) and landed the first reusable time-window primitive that sustained-condition detectors (`phase_locked`, `drifting`, `groove`, ŌĆ”) will compose over. No detector framework, no YAML thresholds, no label schema ŌĆö just a state-level primitive and a regime fixture test.

- **`sustained_windows(condition, min_duration_frames, hysteresis_frames=0)` in `sim/derived.py`.** Takes a 1D bool-castable time series and returns sorted, non-overlapping half-open `[start, end)` frame-index intervals. The interval span (including any absorbed hysteresis gaps) is what gets compared to `min_duration_frames`; the window's endpoints always land on True frames (trailing False hysteresis is never included). `hysteresis_frames=0` disables merging and returns strict contiguous True runs only. Pure function ŌĆö frame counts only, no config / wall-clock / thresholds. Callers convert seconds to frames themselves via `control_rate_hz`. Edge cases verified in-session: empty input ŌåÆ `[]`, all-False ŌåÆ `[]`, all-True ŌåÆ `[(0, T)]` if it meets `min_duration_frames` else `[]`, 2D input rejected, `min_duration_frames < 1` rejected, `hysteresis_frames < 0` rejected, tiny sub-threshold runs dropped silently, mid-run False gaps `> hysteresis_frames` split into separate windows.
- **Interval convention chosen: half-open `[start, end)`.** Rationale: matches Python slicing (`condition[start:end]` yields exactly the window's frames) and NumPy indexing, and `end - start` gives span directly without off-by-one. The alternative (closed `[start, end]`) would have the nicer "last True index is `end`" property but forces every caller to write `+ 1` for length and for slicing ŌĆö worse for the three or four upcoming sustained-condition detectors.
- **Hysteresis semantics chosen: gap-absorption at merge time.** Alternatives considered: (a) count only True frames against `min_duration_frames` (rejected ŌĆö makes "sustained for 3 s" mean "at least 3 s of True out of some larger span", which is a statistical test, not a sustained-condition test); (b) require the condition to hold at both endpoints *and* for all but `hysteresis_frames` interior frames (rejected ŌĆö harder to implement deterministically, and equivalent in most cases to the span-based rule once `min_duration_frames` is comparable to hysteresis). Current rule: a False stretch between two True runs of length `<= hysteresis_frames` is folded into the window; anything longer splits. Simple, deterministic, and the only interior-False-frame count that matters is the one used for merging.
- **DRIFTING sanity test ŌĆö `tests/test_regimes.py::test_drifting_regime_sustains_low_coherence`.** Two checks on the 8 s run of `configs/regime_drifting.yaml`: (1) `sustained_windows(r < 0.5, min=2.0 s, hyst=0.25 s)` returns at least one window, with longest `>= 3.0 s`; (2) tail-2 s mean `r < 0.40`, pinning drifting well below two_cluster's middle band (observed `0.63`) and locked's near-unity (observed `0.995`). Observed: one merged sustained window spanning the full 1599 frames (8.00 s); tail mean `r = 0.2902`; fraction of frames with `r < 0.5` = `0.859`.
- **Regime separation is structural, not numeric.** The same `sustained_windows(r < 0.5, min=2.0 s, hyst=0.25 s)` call returns `[]` on both locked (r stays > 0.95, so `low` is never True) and two_cluster (whose longest merged sub-second valley is 86 frames = 0.43 s, well below the 2.0 s floor). Noted in the test's inline comment ŌĆö the primitive itself is the discriminator, we do not need separate threshold knobs per regime.
- **Existing `configs/regime_drifting.yaml` worked unchanged.** No parameter tuning needed. Tail mean `r Ōēł 0.29` on the same seed/config that the earlier doc-pass-6 cross-check measured at `r Ōēł 0.27`; the small difference is the different tail length (2 s vs 1 s) used for averaging, not a simulator change ŌĆö `state.npz` output is still byte-identical run-to-run.
